# Repository Guidelines

## 프로젝트 구조 및 구성
이 저장소는 정적 웹페이지 기반의 작은 로또 번호 생성기 프로젝트입니다.

- `index.html`: HTML, CSS, JavaScript가 함께 들어 있는 메인 애플리케이션 파일
- `README.md`: 실행 방법과 간단한 기능 설명
- `plan.md`: 작업 계획과 현재 진행 상태 정리
- `.github/`: 필요할 때만 GitHub 관련 설정 파일을 두는 위치

현재는 `src/`, `tests/`, `assets/` 디렉터리가 없습니다. 파일이 커지기 시작하면 `index.html`에 계속 누적하지 말고 스크립트, 스타일, 정적 자산을 분리하세요.

## 빌드, 테스트, 개발 명령
이 프로젝트는 별도 빌드 과정이 필요 없습니다.

- `start index.html`: Windows에서 브라우저로 페이지 열기
- `python -m http.server 8000`: 로컬 정적 서버 실행
- `git status`: 커밋 전 변경 상태 확인

정적 파일만 사용하는 프로젝트이므로, 우선은 브라우저에서 직접 확인하는 방식으로 개발합니다.

## 코딩 스타일 및 네이밍 규칙
HTML, CSS, JavaScript는 기존 파일에 맞춰 공백 2칸 들여쓰기를 사용합니다.

- JavaScript 함수와 변수는 `generateNumbers`, `renderSets`처럼 의미가 분명한 영어 이름 사용
- 사용자에게 보이는 문구는 한국어 유지
- 새 파일 이름은 가능하면 `kebab-case` 사용
- DOM 조작은 명시적으로 작성하고, 프레임워크는 꼭 필요할 때만 도입

현재 포매터나 린터는 설정되어 있지 않습니다. 새로 추가할 경우 설정 파일과 사용 방법을 함께 문서화하세요.

## 테스트 가이드
자동화된 테스트는 아직 없습니다. 변경 후 브라우저에서 수동 검증해야 합니다.

- 번호가 항상 `1~45` 범위인지 확인
- 한 세트가 중복 없는 6개 번호인지 확인
- 빠른 선택, 순차 공개, 재추첨 버튼이 정상 동작하는지 확인

테스트를 나중에 추가한다면 `tests/` 디렉터리를 만들고 `lotto-generator.test.js` 같은 이름 규칙을 사용하세요.

## 커밋 및 PR 가이드
커밋 메시지는 짧고 직접적으로 작성합니다. 이 저장소에서는 한국어 커밋 메시지를 우선 사용합니다.

예시:
- `인터랙션 애니메이션 개선`
- `로또 결과 카드 UI 정리`

Pull Request에는 아래 내용을 포함하세요.

- 사용자 입장에서 무엇이 바뀌었는지 설명
- 변경한 주요 파일 목록
- UI 변경이 있으면 스크린샷 또는 짧은 화면 녹화
- 직접 확인한 테스트 내용
